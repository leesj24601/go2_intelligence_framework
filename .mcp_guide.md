# Isaac Sim MCP 가이드

## 기본 용어

- `stage`: 현재 열려 있는 전체 장면
- `prim`: 장면 안의 개별 노드 또는 객체
- `prim path`: 해당 prim의 주소 같은 경로. 예: `/World/Go2`

## 도구 선택 기준

- 정확한 prim path를 이미 알고 있고 위치, 회전, 스케일만 바꾸면 되는 경우에는 `transform`을 우선 사용한다.
- stage 탐색, 조건 분기, 에셋 경로 탐색, 복잡한 생성 로직이 필요한 경우에만 `execute_script`를 사용한다.

## `transform` 과 `execute_script`의 차이

`transform`은 전용 MCP 명령이다. `prim_path`, `position`, `scale` 같은 구조화된 파라미터를 Isaac Sim extension으로 보내고, extension이 해당 prim에 직접 변환을 적용한다.

`execute_script`는 원격 Python 실행이다. 코드 문자열 전체를 Isaac Sim으로 보내고, 그 안에서 어떤 prim을 찾는지, 어떤 순서로 처리하는지, stage를 어떻게 순회하는지에 따라 결과가 달라진다.

## 왜 `transform`이 더 안정적이었는가

- `transform`은 대상 prim이 명확하다.
- 입력 형식이 고정되어 있다.
- stage 순회나 임시 탐색 로직이 없다.
- 성공/실패 메시지가 더 직접적이다.

이번 작업에서는 `/World/Go2`를 `transform`으로 옮겼을 때는 바로 반영되었고, `execute_script`로 지게차 근처 재배치를 시도했을 때는 눈에 띄는 변화가 없었다. 실제 이유는 스크립트 쪽이 지게차 prim과 Go2 prim을 정확히 찾는 과정에 의존했기 때문이다. 반면 `transform`은 이미 알고 있는 정확한 prim path를 바로 조작했다.

## 실전 규칙

- prim path를 알고 있으면 `transform`을 먼저 쓴다.
- prim을 먼저 찾아야 하면 `execute_script`로 탐색 또는 생성만 수행한다.
- 최종 위치 조정은 다시 `transform`으로 마무리한다.

## 현재 MCP 서버에서 확인된 주의점

이번 세션에서는 로컬 `isaac-sim-mcp` 서버의 `execute_script` 래퍼가 문자열 대신 dict를 반환하는 경우가 있었고, 이 때문에 MCP 응답 검증이 깨졌다. 따라서 `execute_script` 결과 메시지가 이상하다고 해서 Isaac Sim 내부 작업이 반드시 실패했다고 단정하면 안 된다. 먼저 MCP 서버 래퍼의 반환 형식을 확인하는 것이 맞다.
